### 섹션 1: [Markdown 형식의 요약 보고서]

🚀 **프로젝트명:** 마인드 가드너 (Mind Gardener)

🎯 **한 줄 요약 (Executive Summary):**
AI 감성 분석을 통해 사용자의 생각과 감정을 '디지털 정원'으로 시각화하여, 개인의 심리적 안정감('고요한 유능감')을 제공하고, 나아가 익명의 사용자들과 긍정적 감정을 공유하는 집단적 경험('고요한 연결감')을 통해 깊은 유대감을 형성하는 차세대 멘탈 웰니스 플랫폼.

💼 **비즈니스 모델 (Business Model):**
*   **타겟 고객:**
    *   과도한 정보와 업무 스트레스로 '통제 불능감'을 느끼는 30대 지식 노동자.
    *   자신의 감정을 꾸준히 기록하고 성찰하며 내면의 성장을 원하는 현대인.
*   **수익 창출 방식 (대화록 기반 추론):**
    *   대화에서 직접적인 수익 모델은 논의되지 않았으나, 서비스의 특성상 아래 모델이 유력함.
    *   **구독 모델 (Subscription):** 정원의 크기, 저장할 수 있는 씨앗(생각)의 개수, 희귀 아이템 획득 확률 증가 등 프리미엄 기능을 월/년 단위 구독으로 제공.
    *   **프리미엄 기능 판매 (Freemium):** 기본 기능은 무료로 제공하되, 정원을 꾸밀 수 있는 특별한 테마나 아이템을 판매.

🛠️ **핵심 기술 스택 (Tech Stack & Architecture):**
*   **아키텍처:** AWS 클라우드 기반의 마이크로서비스 아키텍처 (MSA) 및 서버리스 우선(Serverless-first) 접근.
*   **프론트엔드:** React Native (+ React Native Skia)
    *   iOS/Android 동시 개발로 빠른 시장 진입.
    *   Skia를 통한 고성능 2D 그래픽 렌더링으로 부드러운 정원 시각화 구현.
*   **백엔드:** Node.js (NestJS) & Python (FastAPI)
    *   핵심 비즈니스 로직은 비동기 처리에 강한 Node.js로 구현.
    *   AI 추론 서비스는 Python 생태계를 활용하여 구축.
*   **데이터베이스:** Amazon DynamoDB (NoSQL)
    *   수억 개의 '씨앗' 데이터를 감당할 수 있는 확장성과 빠른 응답 속도 확보.
    *   GSI(Global Secondary Index)를 활용하여 과거 데이터에 대한 효율적인 쿼리 실행.
*   **AI 모델:** KoBERT (한국어 특화 NLP 모델) & Amazon Comprehend
    *   사용자 텍스트의 감성, 토픽을 분석하여 시각적 요소로 변환.
    *   공유되는 텍스트의 유해성을 검증하는 콘텐츠 모더레이션 기능 수행.
*   **인프라 및 핵심 서비스:**
    *   **AWS Lambda/Fargate:** 서버리스 컴퓨팅 리소스.
    *   **DynamoDB Streams, Amazon Kinesis/Redis:** 사용자 감정의 실시간 집계 및 분석.
    *   **Amazon EventBridge:** 특정 시간/조건에 작업을 실행하는 스케줄러 (기억 되돌아보기, 글로벌 이벤트 감시).
    *   **AWS SNS:** 대규모 사용자 대상 푸시 알림 발송.

⚠️ **잠재적 리스크 및 해결 방안 (Risks & Mitigations):**
*   **리스크 1: 핵심 재미 요소(가변적 보상)의 식상함**
    *   **문제:** 사용자가 '입력(생각) → 결과(꽃/잡초)'의 패턴을 예측하게 되면 흥미가 급격히 감소함.
    *   **해결 방안:** 감성, 키워드, 시간, 사용자 레벨 등 다차원 변수를 조합하는 **'시각적 변환 규칙 엔진'**을 구축하여 예측 불가능한 경이로움을 지속적으로 제공.
*   **리스크 2: 익명 커뮤니티의 콘텐츠 안전성 문제**
    *   **문제:** '별똥별의 밤' 이벤트 시, 익명의 사용자가 작성한 부적절하거나 유해한 텍스트가 공유될 가능성.
    *   **해결 방안:** 공유될 텍스트를 AI 모더레이션(**Amazon Comprehend**) 필터로 사전에 검증하는 **'안전한 생각 공유 파이프라인'**을 구축하여 유해 콘텐츠를 원천 차단.
*   **리스크 3: 글로벌 이벤트 발생 시 서버 부하 폭증**
    *   **문제:** '별똥별의 밤' 시작 시, 푸시 알림을 받은 수많은 사용자가 동시에 접속하여 서버가 다운될 위험.
    *   **해결 방안:** API Gateway의 Throttling, Lambda의 Provisioned Concurrency, DynamoDB의 On-demand Capacity 등 트래픽 급증에 대비한 **서버리스 확장 설계를 적용**하고, AWS SNS를 통해 안정적으로 대규모 알림을 분산 발송.
